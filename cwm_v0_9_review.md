# CWM v0.9 설계 문서 점검 리포트

본 문서는 `CWM_설계문서_v0_9.docx`를 기반으로 설계의 **완성 여부**를 점검한 결과다.  
평가 기준은 **일관성, 모호점 해소 여부, 구현 가능성**이다.

## 1) 요약 결론
- **설계 완성도: 높음 (완료에 근접)**
- **잔여 리스크: 4건**
  문서에 수치화가 들어가며 대부분 해소되었으나, 실제 구현을 위해 **정의/운영 규칙**이 추가로 필요하다.

## 2) v0.9에서 명확히 해결된 항목
- EMA 임계값의 `k` 모호점 → **k=1 단순화**
- 반복 출현 패턴 → **W=20, θ=2로 수치화**
- 문맥 오류 → **문맥 벡터 평균 + 코사인 거리**로 정의
- 곡선별 임계값 비용 → **에폭 단위 + 샘플링 근사**
- EMA 초기값 비용 → **1% 샘플링 기반 추정**
- σ_dim 분리 → **차원 추가 조건 불능 문제 해소**

## 3) 남은 리스크 / 보완 필요 사항
R1. 반복 패턴 “문맥 윈도우” 정의 부족  
- W=20, θ=2는 정의되었으나 **문맥 윈도우의 구성(몇 토큰/몇 턴)**이 불명확

R2. 문맥 벡터 평균의 구성 기준 불명확  
- “활성화된 경로 앵커 평균”이라고 되어 있으나  
  **활성화 기준(강도 임계값, top‑k 등)**이 문서에 명시되지 않음

R3. 곡선별 임계값 샘플링의 안정성  
- 10% 샘플링은 비용을 줄이지만 **표본 분산이 큰 경우 임계값 변동** 가능  
- 최소 샘플 수/안정화 조건 명시 필요

R4. σ_dim 고정 하한의 과도한 차원 증가 위험  
- σ_dim 고정으로 차원 추가가 **너무 잦아질 위험**  
- 차원 증가 빈도 제한(쿨다운 + 최대 증가율) 명시 필요

## 4) 최종 판정
문서 기준으로는 **“사실상 완성”**에 가깝다.  
다만 위 4개 항목은 **구현 난이도 및 안정성에 직접 영향**을 주므로,  
v0.9 문서에 짧은 규칙 정의를 추가하는 것이 권장된다.

## 5) 권장 보완 문구 (문서 삽입용)
1. 반복 패턴 윈도우 정의  
   - “문맥 윈도우는 최근 3턴의 핵심 토큰 시퀀스로 정의한다.”

2. 문맥 벡터 구성 기준  
   - “활성화 강도 상위 20% 앵커의 평균을 문맥 벡터로 사용한다.”

3. 임계값 샘플링 안정화  
   - “곡선별 임계값 샘플 수는 최소 200개를 보장한다.”

4. σ_dim 과도 증가 방지  
   - “차원 추가는 최근 1,000스텝당 최대 1회로 제한한다.”

## 6) 설계 → 구현 체크리스트 (남은 반영 항목)
아래 항목은 v0.9 설계에 포함되어 있으나, **현재 코드에 완전히 반영되지 않은 항목**이다.

1. 대화 곡선 학습 로직
- 질문 → 답변 패턴을 곡선으로 학습하는 규칙이 없음
- 대화 데이터에서 Q/A 전환을 인식하고 별도 곡선으로 강화 필요

2. 문맥 기반 출력 규칙
- 현재 출력은 마지막 토큰 중심
- 문맥 벡터 기반 예측/가중치를 출력 규칙에 직접 반영 필요

3. 문자 임베딩 기반 OOV 처리 완성
- 규칙 기반 OOV 처리가 남아 있음
- 문자 레벨 앵커 학습 및 문자 평균 기반 OOV 초기화 구현 필요

4. 에폭 단위 앵커/메모리 관리
- 병합/삭제는 에폭 단위 규칙이 있으나 실제 에폭 관리 로직이 없음
- “에폭” 정의와 실행 주기 구현 필요

5. 관계/감정/문맥 필터의 실제 생성 반영
- 설계에는 다중 가중치 필터가 있으나
- 현재 생성 로직에서 체감 영향이 약함

6. 반복 패턴 유사성 계산 로직
- W=20, θ=2는 정의되었으나 실제 “윈도우 구성” 규칙이 필요
- 문맥 시그니처 정의와 비교 규칙 구현 필요

7. 곡선별 임계값 샘플링 주기
- 샘플링으로 계산하는 규칙은 있으나
- 구현에서 주기/샘플 크기 고정이 필요

## 7) 권장 진행 순서
1. 문맥 기반 출력 규칙 강화
2. 대화 곡선 학습 로직 추가
3. 문자 임베딩 기반 OOV 처리
4. 에폭 단위 앵커/메모리 관리
5. 관계/감정/문맥 필터 강화
