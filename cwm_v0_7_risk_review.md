# CWM v0.7 설계 문서 리스크 점검 결과 (답변 반영본)

본 문서는 `CWM_설계문서_v0_7.docx`의 내부 일관성, 정의 충돌, 누락된 전제/조건을 중심으로 점검한 결과다. 코드베이스와의 비교나 성능/리소스 검토는 제외했다.

**요약**
문서의 큰 흐름은 일관되지만, 적응형 정의와 동적 확장/삭제 규칙이 동시에 존재해 경계 조건에서 불안정성이 발생할 여지가 있다. 또한 “규칙 없음” 원칙과 일부 고정 계수/조건 사이의 긴장이 남아 있다. 아래 내용은 리스크 지적과 함께 작성자 답변을 반영한 정리본이다.

**구조화 요약 (장·절별 핵심 주장/정의)**
1. 핵심 원칙  
모든 것은 곡선이며 오류 신호로 학습된다. 규칙/하드코딩/고정 임계값을 금지한다. 학습은 국소 역전파로 관련 앵커만 이동한다.

2. 전체 구조 — 곡선 네트워크  
곡선 유형(시각/물리/공간/감정/언어/대화/관계/유머)이 하나의 네트워크로 통합된다. 언어 곡선을 먼저 구현하되, 다른 곡선 좌표는 NULL로 예약한다.

3. Concept State  
개념은 여러 곡선 위 위치의 집합이다. 앵커는 단일 벡터이며 오류 신호로 이동 가능, 루프 계산 중에만 순간 고정된다. NULL 차원은 유사도 계산에서 제외한다.

4. 유사도/거리  
개념 유사도는 코사인, 이동/오류는 유클리드 거리로 정의한다. 사용처별로 동일한 정의를 재사용한다.

5. 위치 업데이트  
새 위치 = P + α(t) × (T - P) × h(dist, σ). α 스케줄링으로 수렴을 보장하고, 진동 방지를 위해 가중 평균 방향을 사용한다. 관련 앵커만 업데이트한다.

6. 오류 신호와 루프 생성  
오류가 임계값보다 크면 새 루프를 생성한다. 임계값은 적응형 정의. 유사도 > 0.9이면 루프 생성 억제.

7. 루프 메커니즘  
루프 위치는 유사도 비율로 결정, 크기는 평균 비유사도로 정의. 기존 앵커는 루프 계산 중에만 고정.

8. Dynamic Dimension  
오류/거리 임계값에 따라 차원 추가, 중요도/상관관계로 압축, 연결 앵커 없음 등으로 삭제한다. NULL 차원은 공통 차원만으로 유사도 계산, 간접 신호로 점진적으로 채운다.

9. 초기화/Cold Start  
앵커는 무작위 초기화로 시작한다. OOV 단어는 글자 유사도로 가장 가까운 앵커 근처에 배치하고 새 루프로 추가한다.

10. 탐색 알고리즘  
단어 앵커에서 국소 탐색만 수행한다. 탐색 신호는 거리 감쇠를 적용하며, 입력마다 자동 결정되는 탐색 중단 임계값을 사용한다.

11. 앵커 개수 관리  
한계 도달 시 병합(유사도 > 0.9), 고립 삭제, 중요도 삭제의 3단계로 관리한다.

12. 반응 생성  
관계/문맥/감정 필터를 동시에 적용해 최종 점수를 산정한다. 문맥은 지나온 경로 자체를 사용한다.

13. 감정 곡선  
좋다/나쁘다 두 극점에서 출발, 감정 가중치로 탐색 점수를 조정한다. 유머는 예측 벗어남으로 정의한다.

14. 메모리 관리  
중요도 기반 압축/삭제를 사용한다. 연결이 많은 곡선은 삭제하지 않는다.

15. 언어 곡선 발달  
설계 철학상 언어는 후발이나 구현 편의상 언어부터 시작하며 NULL 예약으로 충돌을 피한다.

16. 구현 방향  
GRU/Transformer 등 고정 구조 금지, 전역 역전파 금지, 고정 임계값/규칙 기반 형태소 분석 금지. 국소 역전파/적응형 임계값/무작위 초기화는 허용.

**리스크 답변 (영향도/가능성 3단계)**
R1. “규칙 없음” 원칙과 고정 계수(×1.5, ×0.01)  
영향도: 중간 | 가능성: 높음  
답변: 맞는 지적. 고정 계수는 제거해야 한다.  
해결:  
`×1.5` → 오류 분포의 표준편차 기반 자동 결정  
`×0.01` → 신호 감쇠율(σ)에서 자동 계산  
고정 계수 없이 학습 상태 기반 함수로 정의. v0.8에 반영 필요.

R2. Cold Start에서 임계값 민감성  
영향도: 높음 | 가능성: 중간  
답변: 맞음.  
해결: 초기 안정화 구간 도입  
처음 N개 입력은 루프 생성 억제  
N = 전체 어휘의 10% 학습될 때까지  
이후 적응형 임계값 활성화.

R3. 코사인/유클리드 혼합 사용  
영향도: 중간 | 가능성: 중간  
답변: 문제 없음. 역할이 다름.  
코사인 → 방향 비교, 유클리드 → 이동량 계산  
단, 모든 앵커 벡터는 항상 단위 정규화 유지를 문서에 명시.

R4. 활성화 강도 계산식 모호  
영향도: 중간 | 가능성: 높음  
답변: 맞음.  
해결: 활성화 강도 = 초기 신호 강도 × exp(-dist² / 2σ²)  
dist = 1 - 코사인 유사도(입력, 앵커)  
탐색 중단 = 활성화 강도 < 초기 강도 × σ²  
업데이트 크기 = α(t) × 오류 신호 × 활성화 강도

R5. 차원 추가/압축/삭제 충돌  
영향도: 높음 | 가능성: 중간  
답변: 맞음.  
해결: 우선순위 추가 > 압축 > 삭제  
쿨다운: 차원 추가 후 최소 100스텝 삭제 금지  
최소 유지: 추가된 차원은 1에폭 이상 유지

R6. NULL 차원 편향  
영향도: 중간 | 가능성: 중간  
답변: 중요.  
해결: NULL이 채워질 때  
기존 언어 차원 가중치 × 0.9  
새 차원 가중치 × 1.0  
점진적으로 균형을 맞춘다.

R7. 유사도 0.9 기준 다의어 병합 위험  
영향도: 중간 | 가능성: 중간  
답변: 맞음.  
해결: 루프 억제 조건 강화  
코사인 유사도 > 0.9  
AND 최근 문맥 오류 < 임계값  
AND 반복 출현 패턴 유사  
문맥 오류가 크면 새 루프 생성.

R8. 가중 평균으로 분기 구조 손실  
영향도: 중간 | 가능성: 낮음  
답변: 영향 낮음.  
해결: 평균 방향 벡터 분산 계산  
분산 > 임계값이면 다중 모드 감지 → 평균 대신 새 루프 생성

R9. 구조 변경이 수렴 방해  
영향도: 높음 | 가능성: 중간  
답변: 맞음.  
해결: 병합/삭제는 에폭 단위로만 실행  
학습 중간에는 구조 변경 없음  
오류 변화율 < 1%일 때만 실행

R10. OOV 처리가 규칙 기반 위험  
영향도: 중간 | 가능성: 중간  
답변: 맞음.  
해결: 글자 단위 유사도 → 학습된 문자 임베딩으로 대체  
문자 임베딩도 CWM 방식으로 학습 (규칙 없음)

R11. 문맥 경로 길이 스케일 문제  
영향도: 중간 | 가능성: 중간  
답변: 해결 필요.  
해결: 경로 감쇠 도입  
경로 신호 = 원래 강도 × decay^(현재 스텝 - 경로 스텝)  
decay = 0.95

R12. 고정 구조 금지의 경계 모호  
영향도: 중간 | 가능성: 중간  
답변: 명확화 필요.  
금지: 고정된 네트워크 토폴로지 (레이어 수/히든 크기 고정)  
허용: 동적으로 변하는 벡터 차원 (추가/압축/삭제 가능)  
임베딩 차원은 고정 가능하되, Dynamic Dimension으로 변할 수 있어야 한다.

**누락 정의 답변**
Q1. 활성화 강도 계산식  
활성화 강도 = exp(-dist² / 2σ²)  
dist = 1 - 코사인 유사도(입력, 앵커)

Q2. 최근 100개 오류 집계 기준  
입력 단위 기준 (스텝 단위)  
대화 1턴 = 1스텝

Q3. 차원 우선순위와 최소 유지 기간  
우선순위: 추가 > 압축 > 삭제  
최소 유지: 100스텝 (쿨다운)

Q4. 기준 앵커 A, B 선정  
새 개념 C와 코사인 유사도가 가장 높은 상위 2개 앵커

Q5. 문맥 경로의 시간 범위  
decay 기반 자연 소멸  
decay = 0.95^(경과 스텝)  
신호 < 초기의 1% → 경로 제거 (약 90스텝)

Q6. 유사도 0.9 기준 적용 범위  
곡선마다 독립 적용  
시각 곡선과 언어 곡선의 0.9는 다른 공간 기준

Q7. OOV 글자 유사도  
학습된 문자 임베딩 기반  
문자도 CWM 방식으로 앵커화 (규칙 없음)

Q8. 앵커 중요도 감쇠  
중요도 = 활성화 횟수 × decay^(마지막 활성화 이후 경과 스텝)

**테스트 시나리오 답변**
T1. 초기 앵커 0개 상태  
무작위 배치로 시작 → 앵커 0개 상태 존재하지 않음  
첫 입력 전에 전체 어휘 무작위 초기화 완료

T2. 임계값 급격히 낮아질 때  
초기 안정화 구간에서 루프 생성 억제  
이후 EMA 기반 완만한 업데이트로 급변 방지

T3. 차원 추가/삭제 동일 스텝 충돌  
쿨다운 100스텝으로 해결  
추가 직후 삭제 불가

T4. 유사도/거리 혼합 모순  
단위 정규화 항상 유지로 해결  
방향(코사인)과 이동량(유클리드)은 역할이 달라 충돌 없음

**v0.8 반영 체크리스트 (완료 기준 포함)**
R1. 고정 계수 제거  
완료 기준: `×1.5`, `×0.01` 등 고정 계수 삭제 또는 표준편차/σ 기반 자동 계산식으로 대체 문서 명시

R2. 초기 안정화 구간 도입  
완료 기준: “처음 N개 입력은 루프 생성 억제, N은 전체 어휘의 10% 학습될 때까지” 규칙이 문서에 명시

R3. 코사인/유클리드 혼합 정당화  
완료 기준: “모든 앵커 벡터는 단위 정규화 유지” 문구가 위치 업데이트/유사도 정의에 추가

R4. 활성화 강도 공식 명시  
완료 기준: 활성화 강도, dist, 탐색 중단 조건, 업데이트 크기 공식이 모두 문서에 일관되게 기재

R5. 차원 관리 충돌 방지  
완료 기준: 우선순위(추가 > 압축 > 삭제), 쿨다운(100스텝), 최소 유지(1에폭) 규칙이 명시

R6. NULL 차원 편향 완화  
완료 기준: NULL 채움 시 가중치 재조정 규칙(기존 0.9, 신규 1.0)이 명시

R7. 유사도 0.9 병합 보강  
완료 기준: “코사인>0.9 AND 문맥 오류<임계값 AND 반복 출현 패턴 유사” 조건이 문서에 명시

R8. 다중 모드 감지  
완료 기준: 평균 방향 분산 계산 및 임계값 초과 시 새 루프 생성 전환 규칙이 명시

R9. 구조 변경 타이밍  
완료 기준: 병합/삭제는 에폭 단위로만 수행, 오류 변화율 < 1%에서만 실행 규칙이 명시

R10. OOV 규칙 기반 위험 제거  
완료 기준: 문자 임베딩이 학습 기반이고 CWM 방식으로 앵커화됨을 명시

R11. 문맥 경로 감쇠  
완료 기준: decay 기반 감쇠식 및 제거 기준(초기 1% 이하) 명시

R12. “고정 구조 금지” 경계  
완료 기준: 금지/허용 범위(토폴로지 고정 금지, 동적 차원 허용) 구체 문구가 명시
