0.9이면 병합 후보 5. 위치 업데이트 공식 (v0.6~v0.7 완성) CWM의 학습은 역전파가 아니다. 오류 신호로 관련 앵커만 직접 이동시킨다. 기본 공식 새 위치 = P + α(t) × (T - P) × h(dist, σ) 변수 의미 비고 P 현재 앵커 위치 (단일 벡터) 이동 가능 T 실제 다음에 등장한 앵커 위치 (정답) 대화에서는 사용자의 다음 입력 단어의 현재 앵커 위치 α(t) 학습률 스케줄 아래 공식 참조 T - P 오류 방향 벡터 유클리드 방향 dist 현재 앵커와 입력 사이 거리 1 - 코사인 유사도 h(dist, σ) 영향력 감쇠 함수 exp(-dist² / 2σ²) σ 이웃 반경 α와 함께 점점 줄어듦 α 스케줄링 — 수렴 보장 학습 초기엔 크게 이동해서 빠르게 자리잡고, 후기엔 작게 이동해서 정교하게 다듬는다. α(t) = α_초기 × (α_최종 / α_초기)^(t / T) α_초기 = 0.5 ← 학습 시작: 빠르게 자리잡기 α_최종 = 0.01 ← 학습 후기: 정교하게 다듬기 t = 현재 학습 스텝 T = 전체 학습 스텝 진동 방지 — 가중 평균 방향 여러 오류 신호가 동시에 같은 앵커를 다른 방향으로 당길 때 진동이 발생할 수 있다. 가중 평균 방향으로 한 번만 이동해서 해결한다. 평균 방향 = Σ(오류 신호_i × 활성화 강도_i) / Σ(활성화 강도_i) → 충돌하는 신호들의 가중 평균 방향으로 한 번만 이동 → 진동 없이 수렴 보장 국소 역전파 — 관련 앵커만 업데이트 역전파를 금지하는 게 아니라 범위를 제한한다. 방식 역전파 범위 구조 변화 GPU 기존 AI 전체 네트워크 (전역) 없음 — 구조 고정 ✓ CWM v0.7 탐색 시 활성화된 앵커만 (국소) 오류 크면 새 루프 추가 ✓ 관련 앵커 정의 (v0.7 확정) &apos;관련 앵커&apos;가 무엇인지 명확히 정의한다. 별도 규칙 없이 탐색 과정에서 자동으로 선별된다. 관련 앵커 = 탐색 알고리즘(8장) 실행 시 활성화된 앵커 업데이트 크기 = α(t) × 오류 신호 × 활성화 강도 활성화 강도 = 탐색 시 해당 앵커의 신호 강도 (거리 감쇠 적용값) 많이 활성화된 앵커 → 많이 업데이트 조금 활성화된 앵커 → 조금 업데이트 활성화 안 된 앵커 → 건드리지 않음 ※ 탐색이 자동으로 관련 앵커를 선별한다. 별도 규칙 없음. 탐색 알고리즘(8장)과 학습이 자연스럽게 연결된다. 6. 오류 신호 — 루프 생성 결정 5장(오류 신호)이 루프를 만들지 결정하고, 7장(루프 메커니즘)이 어디에 만들지 결정한다. 두 장은 순서대로 작동하며 충돌하지 않는다. 오류 신호 오류 신호 = T - P_predicted = (실제 다음 앵커 위치) - (예측한 다음 앵커 위치) T = 실제 다음에 등장한 단어의 현재 앵커 위치 대화에서는 사용자의 다음 입력 단어의 앵커 위치 루프 생성 조건 오류 = |T - P_nearest| ← 유클리드 거리 if 오류 &gt; 오류 임계값: → 새 루프 생성 (7장에서 위치 결정) else: → 가장 가까운 앵커를 T 방향으로 당김 (위치 업데이트) 오류 임계값 — 적응형 (v0.7 확정) 고정 숫자를 쓰면 규칙이 된다. 데이터에서 자동으로 결정한다. 오류 임계값 = mean(최근 100개 오류) × 1.5 학습 초기: 오류가 크므로 임계값도 큼 → 루프 덜 생성 학습 후기: 오류가 작아지므로 임계값도 작아짐 → 정교한 루프 루프 생성 억제 조건 (v0.7 신규) 앵커가 무한정 늘어나는 것을 방지한다. 새 입력과 기존 앵커의 최대 코사인 유사도 &gt; 0.9 → 충분히 비슷한 앵커가 이미 있음 → 새 루프 생성 안 함, 기존 앵커 업데이트만 7. 루프 메커니즘 — 루프 위치 결정 6장이 루프를 만들지 결정하면, 이 장이 어디에 어떤 크기로 만들지 결정한다. 루프 위치 공식 A까지 거리 = 1 - 유사도(C, A) ← 코사인 유사도 기반 B까지 거리 = 1 - 유사도(C, B) C의 위치 = 유사도 비율로 결정 (유사할수록 가깝게) 루프 크기 = 평균 비유사도 순간적 고정 (v0.6에서 확정) 기존 앵커는 영구 고정이 아니다. 루프 위치 계산하는 동안만 순간적으로 고정된다. 새 개념 C 들어올 때: 1. 기준 앵커 A, B를 순간 고정 2. 루프 위치 계산 3. 계산 완료 후 A, B도 오류 신호로 이동 가능 ※ &apos;기존 점은 절대 이동하지 않는다&apos;는 v0.4 설계 문구는 루프 계산 중 순간 고정을 의미했다. 영구 고정이 아니다. 영구 고정이면 학습이 불가능하고 CWM 철학(모든 것은 오류 신호로 학습)에도 위반된다. 8. Dynamic Dimension 차원 추가 조건 예측 오류 &gt; 임계값 AND 근접 개념과 거리 &lt; 임계값 → 차원 1개 추가 차원 관리 3단계 (v0.6~v0.7 완성) 단계 조건 동작 추가 오류 &gt; 임계값 AND 근접 개념과 거리 &lt; 임계값 차원 1개 추가 압축 중요도 &lt; 전체 하위 10% AND 다른 차원과 상관관계 높음 여러 차원을 하나로 합침 (세부는 잃지만 핵심 유지) 삭제 압축 후에도 중요도 하위 10% AND 연결 앵커 없음 차원 제거 차원 중요도 = 해당 차원에서 유사도 계산에 기여한 횟수 차원 임계값 = 전체 차원 중요도의 하위 10% (적응형) NULL 처리 — 간접 경로 기존 개념의 새 차원 = NULL (아직 모름) 유사도 계산 = 공통 차원만 사용 (NULL 차원 제외) 간접 경로로 신호 전파: 거리마다 절반씩 약해짐 간접 신호가 NULL을 초기값으로 깨움 이후 직접 학습으로 정교화 9. 초기화 — Cold Start 해결 (v0.7 확정) 학습 시작 시 앵커가 하나도 없는 상태에서 어떻게 시작하는가. 초기 앵커 배치: 무작위 처음 앵커 위치를 무작위로 배치한다. 이유: 어차피 오류 신호로 교정됨 — 초기 위치는 중요하지 않음 동시 등장 빈도 기반 초기화는 규칙 = CWM 철학 위반 설계 원칙 &apos;처음엔 틀려도 경험으로 교정됨&apos; (10장)과 일치 초기화: 모든 앵커 벡터를 균등 분포 무작위로 설정 범위: [-0.1, 0.1] ← 너무 크면 초기 오류가 극단적 어휘에 없는 단어 처리 입력 단어가 어휘집에 없으면: 1. 글자 단위 유사도로 가장 가까운 앵커 찾기 예: &apos;사과들&apos; → &apos;사과&apos; 앵커 근처에 배치 2. 새 루프로 추가 3. 이후 경험으로 정교화 ※ Cold Start 문제는 무작위 초기화로 해결된다. 완전히 빈 상태는 존재하지 않는다. 첫 입력부터 탐색 가능하며, 초기에는 엉뚱한 결과가 나와도 경험으로 점점 교정된다. 10. 탐색 알고리즘 전체 탐색 금지. 단어 앵커에서 국소 탐색만 수행. 탐색 순서 1. 입력 문장에서 단어 추출 &apos;오늘 밥 먹었어?&apos; → [오늘, 밥, 먹다] 2. 각 단어의 곡선 위치로 즉시 이동 (앵커 포인트) 3. 앵커에서 간접 경로로 주변 탐색 신호 = 초기 강도 × exp(-dist² / 2σ²) 신호 &lt; 초기 강도 × 0.01 이면 탐색 중단 ← 적응형 임계값 4. 활성화된 영역에서 가장 가까운 앵커 선택 → 그게 출력 단어 탐색 중단 임계값 — 적응형 (v0.7 확정) 탐색 중단 임계값 = 초기 신호 강도 × 0.01 → 처음 신호의 1% 이하로 약해지면 중단 → 간접 경로 감쇠(거리마다 절반)와 자연스럽게 연결 → 고정 숫자 아님 — 입력마다 자동 결정 곡선은 연속 공간, 출력은 가장 가까운 앵커 포인트 (단어) 문장 = 앵커 포인트들의 순서 별도 변환 장치 불필요 11. 앵커 개수 관리 (v0.7 신규) 루프가 계속 추가되면 앵커가 무한정 늘어난다. 메모리 관리(12장)와 동일한 원리로 해결한다. 앵커 중요도 앵커 중요도 = 해당 앵커가 탐색에서 활성화된 횟수 = 선의 굵기와 동일한 개념 앵커 수 관리 3단계 단계 조건 동작 병합 앵커 수 한계 도달 AND 두 앵커의 코사인 유사도 &gt; 0.9 가까운 앵커 여러 개 → 중간 위치 하나로 병합 고립 삭제 병합 후에도 부족 AND 연결된 앵커 없음 고립된 앵커 삭제 중요도 삭제 여전히 부족 중요도 가장 낮은 앵커 삭제 ※ 앵커 개수 관리는 메모리 관리(12장)와 완전히 같은 원리다. 설계 일관성이 유지된다. 12. 반응 생성 — 다음 위치 선택 두 가지 필터 동시 적용 필터 1 — 관계 곡선 상대방이 친구 → 친구 곡선 활성화 → 반말 앵커 강조 필터 2 — 문맥 곡선 이전 대화 경로 → 문맥 곡선 활성화 → 연결되는 앵커 강조 최종 점수 = 기본 유사도 점수 × 관계 곡선 가중치 × 문맥 곡선 가중치 × 감정 가중치 (13장) 지나온 경로 자체가 문맥 (별도 저장 불필요) 중요한 내용은 반복 활성화로 굵어짐 → 장기 유지 이산적 사실 (이름, 날짜)만 최소한으로 저장 대화 끝나면 사실 정보 삭제 13. 감정 곡선 출발점: 좋다/나쁘다 두 극점. 태어날 때부터 존재. 초기: 좋다 ←────────────→ 나쁘다 경험 축적 배고픔 → 나쁘다 쪽으로 당김 따뜻함 → 좋다 쪽으로 당김 복잡한 감정 = 두 극점의 조합 슬픔 = 소중함(좋다) + 잃어버림(나쁘다) 동시 활성화 감정과 탐색의 연결 (v0.5에서 확정) 기본 점수 = 코사인 유사도(입력, 앵커) 감정 가중치 = 감정 곡선 위치 (좋다 쪽 → +, 나쁘다 쪽 → -) 최종 점수 = 기본 점수 × (1 + 감정 가중치 × 강도) 기쁜 상태 → 좋다 쪽 앵커들이 더 강하게 활성화 슬픈 상태 → 나쁘다 쪽 앵커들이 더 강하게 활성화 중립 상태 → 감정 가중치 ≈ 0, 순수 유사도로만 탐색 감정 업데이트도 위치 업데이트 공식(5장) 동일하게 적용 유머 처리 정상 문장 = 활성화 영역 안의 앵커 선택 농담 = 의도적으로 활성화 영역 밖의 앵커 선택 웃음 = 예상 벗어남의 크기 유머 감각도 경험으로 학습되는 곡선 14. 메모리 관리 삭제 기준 — 시간이 아니라 중요도 선의 굵기 = 사용 빈도 = 중요도 = 생존 기준 단계 조건 동작 1단계 메모리 한계 도달 얇은 곡선들 압축 (여러 개 → 요약 하나) 2단계 압축 후에도 부족 고립된 곡선 삭제 (연결 없는 것) 3단계 여전히 부족 압축된 요약 중 가장 얇은 것 삭제 완전 삭제 전에 압축 먼저 압축 = 세부는 잃지만 핵심은 유지 연결 많은 것은 절대 삭제 안 됨 인간 뇌의 망각 메커니즘과 동일 15. 언어 곡선 발달 순서 언어 곡선은 처음부터 있는 게 아니다. 다른 곡선들이 형성된 후 반복 동시 활성화로 자연스럽게 생겨난다. 단계 상태 내용 1단계 (태어날 때) 시각 곡선만 존재 픽셀 차이로 형태 구분 시작. &apos;이건 둥글다, 이건 뾰족하다&apos;
